## 1. 引言

### 1.1 项目概述

**AI模型训练管理平台** 是一个基于 Flask 的 Web 应用，提供从数据上传、模型训练、超参数调优、模型评估、预测推理到 Docker 部署的完整 AI 工作流。用户无需编写代码，通过浏览器即可完成端到端的机器学习任务。

### 1.2 当前状态与关键指标

| 指标 | 数值 |
|------|------|
| 已训练模型 | 114 个 (68 NLP / 46 通用) |
| 数据集 | 100 个 (15 公开数据源 / 11 类别感知) |
| 训练框架 | 5 种 (sklearn / PyTorch / TensorFlow / Transformers / ONNX) |
| API 端点 | 60+ 个 (7 个 Tag 分组，Swagger 3.0.3 文档) |
| 自动化测试 | 356 个 (pytest, 覆盖率 73%) |
| 质量治理脚本 | 5 个 (诊断/清理/重训练/定向重训/看门狗) |
| 支持任务类型 | 分类 / 回归 / 聚类 / NLP / 计算机视觉 / 生成式 / 强化学习 |
| 模型质量 | 0 过拟合 (acc≥0.999)、0 常数预测器 |

### 1.3 术语定义

| 术语 | 定义 |
|------|------|
| **训练任务 (TrainingJob)** | 一次完整的模型训练执行记录 |
| **模型记录 (ModelRecord)** | 训练产出的模型文件 + 元数据（准确率、框架等） |
| **数据集 (Dataset)** | 用于训练或评估的结构化数据文件 (CSV/Excel/JSON/Parquet) |
| **训练器 (Trainer)** | 框架特定的训练执行器 (SklearnTrainer / PyTorchTrainer / KerasTrainer / TransformersNLPTrainer) |
| **超参数调优** | GridSearchCV / RandomSearchCV / AutoML 自动搜索最佳参数组合 |
| **Tier 分级** | 模型质量三级分类：Tier1(必删) / Tier2(高风险) / Tier3(警告) |
| **SSE** | Server-Sent Events — 服务器推送实时训练进度 |
| **TF-IDF** | Term Frequency-Inverse Document Frequency — 文本特征提取 |
| **模型打包 (Bundle)** | 包含模型对象 + Scaler + Vectorizer + LabelEncoder 的 pickle 字典 |

#### 3.5.2 预测管道 (Inference Pipeline)

```
输入 (文本/特征/文件/图像)
  │
  ▼
ModelInferenceService.load_model()        ← 加载 bundle (.pkl/.pt/transformers)
  │  ├── sklearn: pickle.load → model, scaler, vectorizer, label_encoders
  │  ├── PyTorch: torch.load + config
  │  └── Transformers: AutoModel.from_pretrained + AutoTokenizer
  ▼
predict(model, DataFrame)
  │  ├── 特征对齐 (补充缺失列, 去除多余列)
  │  ├── 缺失值填充 (数值 mean / 分类 mode)
  │  ├── 分类特征编码 (LabelEncoder, 未知类别 fallback)
  │  ├── 数值标准化 (StandardScaler, 跳过 tfidf_* 列)
  │  ├── 框架分发预测
  │  │    ├── sklearn: .predict() + .predict_proba()
  │  │    ├── PyTorch: model(tensor).softmax()
  │  │    └── Transformers: tokenize → model(**inputs) → softmax
  │  ├── 标签解码 (LabelEncoder.inverse_transform)
  │  └── 概率提取 (Top-3 类名+概率)
  ▼
输出 {success, predictions[], probabilities[], task_type, num_samples}
```

**Tier 分级标准**:

| Tier | 标签 | 条件 | 动作 |
|------|------|------|------|
| Tier 1 | definite_delete | 常数预测器 OR acc=1.0 | 自动删除 |
| Tier 2 | high_risk_delete | acc≥0.999 OR F1差距>0.15 OR 训练测试差距>0.15 AND acc≥0.95 | 审查后删除 |
| Tier 3 | warning_only | 健康评分<40 OR 其他警告 | 标记跟踪 |
| Healthy | — | 无上述问题 | 保留 |

## 5. API 接口规范

### 5.1 认证方式

所有 `/api/v1/` 端点需要认证，支持三种方式 (优先级递减):

1. **Session Cookie**: Web 浏览器登录后自动携带
2. **JWT Bearer Token**: `Authorization: Bearer <access_token>` (有效期 7200 秒)
3. **API Key**: `X-API-Key: <user_api_key>` 或 `?api_key=<key>`

### 5.2 通用响应格式

**成功响应**:
```json
{
  "success": true,
  "data": { ... }
}
```

**错误响应**:
```json
{
  "success": false,
  "message": "错误描述",
  "error_code": "ERROR_CODE"
}
```

### 5.3 端点目录 (7 Tag 分组)

| Tag | 端点数 | 说明 |
|-----|--------|------|
| Auth | 4 | 登录/注册/刷新令牌/用户信息 |
| Datasets | 8 | CRUD + 公开导入 + 预览 + 自动配置 |
| Models | 18 | 列表/详情/创建/删除/下载/导出/模型卡 |
| Prediction | 9 | predict/quick-predict/serve/predict-export/template/evaluate/feature-importance |
| Training | 6 | 创建/控制(暂停/取消/恢复)/状态/日志 |
| Hyperparameter | 5 | GridSearch/RandomSearch/AutoML/进度/结果 |
| Dashboard | 4 | 总览/排行榜/统计/最近活动 |

完整 Swagger 文档: http://127.0.0.1:5000/apidocs/

### 5.4 预测 API 详解

#### quick-predict (推荐用于单样本测试)

```
POST /api/v1/models/<uuid>/quick-predict
Content-Type: application/json
Authorization: Bearer <token>

请求体 (NLP 文本):
{"text": "这部电影非常精彩，强烈推荐！"}

请求体 (特征键值对):
{"features": {"feature1": 1.5, "feature2": 2.3, ...}}

响应:
{
  "success": true,
  "data": {
    "prediction": "正面",
    "confidence": 0.927,
    "probabilities": [
      {"class": "正面", "probability": 0.927},
      {"class": "负面", "probability": 0.073}
    ],
    "model_type": "nlp",
    "input_mode": "text",
    "input_preview": "这部电影非常精彩，强烈推荐！",
    "num_samples": 1
  }
}
```

#### predict (批量预测)

```
POST /api/v1/models/<uuid>/predict

文件上传: CSV/Excel/JSON 文件作为 file 字段

响应:
{
  "success": true,
  "data": {
    "predictions": ["正面", "负面", "正面", ...],
    "probabilities": [
      [{"class": "正面", "probability": 0.85}, ...],
      ...
    ],
    "task_type": "classification",
    "num_samples": 100
  }
}
```

### 6.2 核心表结构

**User** (用户): id, username, email, password_hash, role{admin/researcher/viewer}, api_key, token_version, is_active

**Dataset** (数据集): id, name, owner_id(FK→User), file_path, file_format{csv/xlsx/json/parquet/txt}, file_size, row_count, column_count, category{nlp/cv/tabular/...}, status, is_public, summary_json

**TrainingJob** (训练任务): id, user_id(FK→User), dataset_id(FK→Dataset), model_id(FK→ModelRecord), name, framework, algorithm, ml_task_type, total_epochs, current_epoch, status{running/paused/completed/failed/cancelled}, progress, log_text, started_at, completed_at

**ModelRecord** (模型记录): id, uuid(唯一), name, framework{sklearn/pytorch/tensorflow/transformers/onnx}, model_type{nlp/classification/regression/clustering/cv/...}, model_file_path, hyperparameters_json, metrics_json, accuracy, precision, recall, f1_score, status{trained/failed/deploying/deployed}, owner_id(FK→User), training_job_id(FK→TrainingJob), created_at

### 7.2 看门狗机制

```
quality_watchdog.py
├── --once          单次扫描 → JSON 报告 (兼容 clean_overfit_models.py)
├── --daemon        后台守护 (threading.Event.wait 模式, 同 cache.py)
├── --quick         快速模式 (仅常数预测器检查，跳过健康评分)
├── --auto-clean    自动删除 Tier 1 模型 (需 --force 确认)
├── --since DATE    增量扫描 (仅检查日期后更新的模型)
└── --interval N    守护模式扫描间隔 (默认 1440 分钟 = 每日)
```
